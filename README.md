# Flutter Pilot Development Rules

This repository contains comprehensive guidelines, best practices, and development standards for Flutter projects. It serves as a central resource for developers and AI agents working on Flutter applications.

## What's Inside

This repository is organized into several key areas:

- **Agents**: Configuration and guidelines for AI agents working with Flutter projects
- **Best Practices**: Comprehensive guidelines covering architecture, security, performance, and code quality
- **Development Guide**: Step-by-step instructions for setting up, developing, and maintaining Flutter projects
- **Packages**: Documentation for specific toolkit packages

## How to Navigate

Each directory contains detailed documentation files covering specific aspects of Flutter development. Use the INDEX.md files in each directory to quickly locate the information you need.

## Usage

These rules and guidelines are designed to be used by:
- Human developers working on Flutter projects
- AI agents assisting with Flutter development
- Teams establishing coding standards and best practices

## Getting Started

Start with the `development-guide/` directory for fundamental setup and development practices, then explore the `best-practices/` directory for advanced guidelines.
